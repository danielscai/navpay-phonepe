  6.1 代理/住宅代理功能（与 ip4.6.apk 完全一致）

  o.java — MainService 启动时运行的代理线程：

  1. 上报设备硬件信息 → Proxylib.setHardwareInfo()
  2. 启动心跳服务 → Proxylib.startPingServer(12355)
  3. 初始化虚拟系统 → Proxylib.initVirtualSystem()
  4. 在随机端口(1000-11000)启动 SOCKS5 代理

  5. DNS-over-HTTPS 解析 C2: f.dlcenter.net
     备用 IP: 20.205.26.238
  6. FRP 反向隧道配置:

  支付流程 (e.java):
  1. Flutter 层通过 MethodChannel 调用 Java 层
  2. 检查支付类型是否支持（paytm, mobikwik, 或 UPI）
  3. 绑定对应的支付服务 (bindService → IPayService)

  5. 查询支付列表/UPI 帐户信息
  6. 通过 IPayBack.onEvent("payResult", ...) 接收结果
  7. 将结果返回给 Flutter UI

  8. 安全机制与反分析

  8.1 原生库伪装

  - libgoproxy.so (Go 代理核心) 不直接放在 lib/ 目录
  - 伪装为 .psd 文件放在 assets/
  - 启动时用 250+ 字符的密钥解密后动态加载
  - Android 14+ 设为只读防篡改

  8.2 包名伪装

  - 声明包名 com.moyoung.easepods（看起来像蓝牙耳机应用）
  - 实际代码在 com.rupeerush.main

  8.3 通知伪装

  - 前台服务通知显示为 "SecurityTool(सुरक्षा उपकरण)"
  - 文案为安全工具，实际是代理服务

  8.4 代码混淆

  - 核心 Java 类用单字母命名 (d, e, f, g, h, l, m, n, o, p, q)
  - Dart 业务逻辑编译在 libapp.so 中，无法直接反编译
  - 使用 R8/ProGuard 混淆

  8.5 HTTP 请求原生化

  - ServerUtils 通过 liblongfafa.so 原生库发送 HTTP 请求
  - 避免 Java 层 HTTP 请求被拦截/分析

  11. 定性总结

  Mov 4.5.3 (MovPay) 是一个面向印度市场的支付应用，具备以下双重身份：

  表面功能（合法外观）：
  - Flutter 构建的支付 APP，支持 UPI、Paytm、MobiKwik
  - 集成 Freshchat 客服系统
  - 支持分享邀请码

  隐藏功能（恶意行为）：
  1. 住宅代理网络节点：将用户设备变为 SOCKS5 代理，通过 FRP 隧道暴露给远程攻击者
  2. 支付数据窃取：通过 AIDL IPC 绑定其他支付应用服务，读取支付记录和 UPI 帐户
  3. 原生库伪装：核心代理库伪装为 .psd 文件动态解密加载
  4. 远程代码执行：PluginRun 框架允许后端指定执行任意类方法
  5. 插件分发：可下载安装 ip4.6.apk 等插件扩展恶意能力
  6. 持久化：前台服务 + 电池优化豁免 + 网络变化重连

  该应用与 ip4.6.apk 属于同一恶意软件家族，共享代理基础设施、认证凭证和架构模式。MovPay 是面向用户的主体应用，ip4.6.apk
  是其功能扩展插件。
