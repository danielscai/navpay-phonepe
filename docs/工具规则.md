# 工具脚本规范（inject/test）

## 目标
统一各模块脚本入口，保证 build-orchestrator 可统一调用，同时保留模块差异实现。

## 目录与命名
- 所有模块脚本统一放在：`src/{app}/scripts/`
- 标准脚本名称：
  - `inject.sh`：注入脚本
  - `test.sh`：已废弃（测试统一由 build-orchestrator 管理）

## 参数约定
- `inject.sh`：
  - 用法：`./inject.sh <decompiled_apk_dir>`
- `test.sh`：
  - 已废弃，使用 `build-orchestrator` 统一测试逻辑

## build-orchestrator 规则
- `cache_manifest.json` 中必须配置：
  - `inject_script`: 指向模块的 `scripts/inject.sh`
  - `test_mode`: 统一使用 `unified`
- build-orchestrator 只负责流程编排：plan → pre-cache → build-modules → inject → compile → test
- inject 细节逻辑由各模块脚本独立实现
- test 逻辑统一由 build-orchestrator 实现

## 兼容性
历史脚本路径（如 `tools/patch_https_interceptor.sh`、`src/apk/signature_bypass/tools/merge.sh`）保留为薄封装，内部转调到新的 `scripts/inject.sh`。

## 手动测试
由于 test 时间较长，建议手动运行：
- `yarn orch:test:sigbypass`
- `yarn orch:test:https`
- `yarn orch:test:phonepehelper`

> 注意：测试依赖 adb 设备、签名 APK 以及有效的应用包名/启动 Activity。

## 稳定性结论

- 结论：测试阶段必须先卸载旧包再安装新包，否则可能出现“偶现失败”。
- 现象：不卸载时，第一次启动/日志检查可能失败；卸载后再安装则稳定通过。
- 对应实现：`build-orchestrator` 的 `unified_test` 在安装前执行 `adb uninstall`。
