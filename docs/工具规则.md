# 工具脚本规范（inject/test）

## 目标
统一各模块脚本入口，保证 cache-manager 可统一调用，同时保留模块差异实现。

## 目录与命名
- 所有模块脚本统一放在：`src/{app}/scripts/`
- 标准脚本名称：
  - `inject.sh`：注入脚本
  - `test.sh`：测试脚本

## 参数约定
- `inject.sh`：
  - 用法：`./inject.sh <decompiled_apk_dir>`
- `test.sh`：
  - 用法：`./test.sh <signed_apk> <package> <activity> [serial]`

## cache-manager 规则
- `cache_manifest.json` 中必须配置：
  - `inject_script`: 指向模块的 `scripts/inject.sh`
  - `test_script`: 指向模块的 `scripts/test.sh`
  - `test_mode`: 统一使用 `script`
- cache-manager 只负责流程编排：pre-cache → inject → compile → test
- inject/test 细节逻辑由各模块脚本独立实现

## 兼容性
历史脚本路径（如 `tools/patch_https_interceptor.sh`、`src/signature_bypass/tools/merge.sh`）保留为薄封装，内部转调到新的 `scripts/inject.sh`。

## 手动测试
由于 test 时间较长，建议手动运行：
- `src/signature_bypass/scripts/test.sh`
- `src/https_interceptor/scripts/test.sh`
- `src/phonepehelper/scripts/test.sh`

> 注意：test 脚本依赖 adb 设备、签名 APK 以及有效的应用包名/启动 Activity。
