## 1. 概述

### 1.1 背景
合并后的 PhonePe APK 使用 debug keystore 重新签名后，签名与原始签名不同，导致应用自检失败。

### 1.2 解决方案
使用 Pine Hook 框架在运行时 Hook `PackageManager.getPackageInfo()` 方法，当应用查询自身签名时，返回原始 PhonePe 签名而非实际的 debug 签名。

## 附录：签名绕过原理

### A.1 原始签名存储位置

```
smali_classes14/com/PhonePeTweak/Def/HookUtil.smali
```

包含两个静态字符串字段：
- `Signature`: 原始 PhonePe 签名 (用于 GET_SIGNATURES)
- `Certificates`: 原始 PhonePe 证书 (用于 GET_SIGNING_CERTIFICATES)

### A.2 Hook 触发条件

| PackageManager 方法 | flags 值 | 处理逻辑 |
|--------------------|---------|---------|
| getPackageInfo(pkg, 64) | 0x40 | 替换 packageInfo.signatures[0] |
| getPackageInfo(pkg, 0x8000000) | 0x8000000 | 替换 signingInfo.apkContentsSigners[0] |

### A.3 Hook 调用链

```
应用启动
  ↓
PhonePeApplication.attachBaseContext()
  ↓
Plugin.attach(context)
  ↓
PineHelper.start(context)
  ↓
Pine.hook(getPackageInfo, PineHelper$1)
  ↓
当应用调用 getPackageInfo() 时
  ↓
PineHelper$1.afterCall() 拦截并替换签名
```
